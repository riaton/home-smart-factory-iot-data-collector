# 要件定義書

## Home Smart Factory -- IoT設備監視基盤

------------------------------------------------------------------------

# 1. 概要

## 1.1 背景

製造業では、設備の稼働状況や環境情報をデータとして収集し、可視化・分析することで、生産効率向上や異常検知を行う取り組みが進んでいる。\
本プロジェクトでは、家庭環境を小型の工場に見立て、IoTデバイスから取得したデータをクラウドへ集約し、可視化・分析するシステムを構築する。

本システムは以下の技術要素の実証を目的とする。

-   IoTデバイスによるデータ取得
-   クラウド上でのデータ収集・保存
-   設備監視ダッシュボードの構築
-   日次レポート生成
-   閾値ベース異常検知
-   ユーザーごとの設定管理
-   Googleログインによる認証

------------------------------------------------------------------------

## 1.2 目的

本システムの目的は以下とする。

-   家庭内デバイスからIoTデータを収集する
-   AWS上にデータ収集基盤を構築する
-   データを時系列で保存する
-   ECS上でデータ処理（保存/異常検知/集計）を実行する
-   Grafanaで設備監視ダッシュボードを提供する
-   Reactベース画面で認証・レポート・異常一覧・設定管理を提供する
-   ユーザーごとに異常検知条件を設定可能にする
-   前日分データをもとに日次レポートを自動生成する
-   Redisを使ってセッション管理を行う

------------------------------------------------------------------------

# 3. データ要件

## 3.1 取得データ

  項目          説明
  ------------- ----------------------
  device_id     デバイス識別子
  user_id       ユーザー識別子
  temperature   温度
  humidity      湿度
  motion        人感センサー検知結果
  power_w       消費電力
  timestamp     データ取得時刻

## 3.2 データ形式

``` json
{
  "user_id": "user_001",
  "device_id": "room01",
  "temperature": 24.3,
  "humidity": 58,
  "motion": 1,
  "power_w": 120,
  "timestamp": "2026-03-14T21:00:00"
}
```

## 3.3 データ保持期間

| データ種別 | 保持期間 |
|---|---|
| IoTデータ（センサー生データ） | 90日 |
| 異常検知ログ | 1年 |
| 日次レポート | 1年 |

# 4. 主要機能

## 4.1 IoTデータ収集

IoTデバイスから取得したデータをJavaデータ収集クライアントで収集し、
AWS IoT Coreへ送信する。

## 4.2 データ保存

ECSデータ処理ワーカーがSQSから受信データを取得し、RDS(PostgreSQL)へ保存する。

## 4.3 ダッシュボード表示

Grafanaで以下の情報を可視化する。

-   温度推移
-   湿度推移
-   消費電力推移
-   在室状態

GrafanaはIPホワイトリスト（sg-grafana）＋Google OAuth で保護された管理者専用ダッシュボード。一般ユーザー向けのデータ可視化はReact（ダッシュボード画面）で提供する。

------------------------------------------------------------------------

# 5. 日次レポート機能

## 5.1 レポート生成

前日00:00〜23:59のデータを集計し、毎日1回レポートを生成する。
レポート生成はEventBridgeのスケジュールによりECS定期バッチで実行する。

## 5.2 レポート内容

-   総電力消費量
-   デバイス別消費電力量
-   現在温度
-   現在湿度
-   総在室時間
-   異常検知件数
-   異常検知内容

## 5.3 レポートダウンロード

ユーザーは画面からレポートをダウンロードできる。

- ダウンロード形式: PDF
- 制約: 1ユーザーあたり1日最大3回までダウンロード可能
- リセット: ダウンロード回数は毎日0時にリセットする

------------------------------------------------------------------------

# 6. 異常検知機能

閾値ベースで異常を検知し、異常検知ログを保存する（ECSデータ処理ワーカーで実行）。

## 6.1 検知対象

-   総消費電力量の閾値超過
-   温度上限 / 下限超過
-   湿度上限 / 下限超過

ユーザーは設定画面から閾値を自由に設定できる。

## 6.2 異常通知

異常を検知した場合、Amazon SNSを通じて該当ユーザーのメールアドレスに通知する。

- 通知先: ユーザーがGoogleアカウントでサインアップした際に登録されたメールアドレス
- 通知タイミング: 異常検知時に即時送信
- 通知内容: デバイスID、検知内容、検知時刻、閾値、実測値

------------------------------------------------------------------------

# 7. 認証機能

Googleログインを利用する。

-   OAuth認証
-   複数ユーザー対応
-   全ユーザー同一権限
-   初回ログイン時にユーザー情報（user_id、メールアドレス）をRDSへ登録する
-   セッション管理はRedis（ElastiCache）で行う

------------------------------------------------------------------------

# 8. デバイス・ユーザー管理

## 8.1 デバイスとユーザーの関係

- 1ユーザーが複数デバイスを保有できる
- デバイスはuser_idによりユーザーと紐付く
- 異常検知・レポート集計はユーザー単位で行い、デバイス別の内訳も保持する
- 1デバイスを複数ユーザーで共有するケースは対象外とする

------------------------------------------------------------------------
